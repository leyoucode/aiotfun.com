aiotfun.com 上线第一版的时候，是一个 **AI Agent 自动运营的 AIoT 资讯站** —— 爬 RSS、自动评分、自动写稿、自动发布。骨架挺好玩，最后没继续走那条路。这篇记一下我为什么把它拆了重做。

## 资讯站这条路的问题

不是技术问题。技术上跑得挺顺：每天能稳定出 4-6 篇 AI 总结过的内容，去重和打分的链路也跑得动。但越跑越不对劲：

- **没有"我"**：聚合的内容都是别人造的发现，我做的只是"重新写一遍"。读者也好、招聘方也好，看完一篇不知道作者是谁。
- **AI 写作的天花板很低**：AI 能整合信息，但写不出"我熬了 3 个晚上发现 ESP32 这个引脚不能这么接"那种东西。后者才是真正有价值的内容。
- **新闻的半衰期太短**：今天写完的"重磅发布"，3 个月后没人会回来看。但一个折腾笔记可能 2 年后还有人 Google 到。

## 我真正想要什么

想清楚之后发现：我想要的不是又一个内容站，而是**一个能装下我所有项目和折腾的地方**。

- 我做过 8+ 个 AI / IoT 项目，但散落在 GitHub 各处，对外几乎不存在 —— 这是个浪费。
- 我经常在调试时踩到一些有趣的坑，平时记在飞书里给自己看，从没整理给别人看。
- 我对 AI + IoT 的兴趣是真实的，跟 aiotfun 这个域名也匹配 —— 只是主语该是"我做了什么"，不是"我聚合了什么"。

## 重做的几个原则

- **保留域名 aiotfun.com**：虽然这个名字带"Fun"，看起来不像个人品牌，但我不打算花钱买新域名。AI + IoT + Fun 三个词，刚好概括我想折腾的方向。
- **保留所有经过验证的技术栈**：Astro / Tailwind / MDX / 双语 / 暗色 / Pagefind / Giscus / Cloudflare 这些都不动，只换信息架构。
- **删掉所有为"自动化"服务的东西**：collector、AI Agent 团队展示、Newsletter、6 大固定 category、周刊 —— 一刀切。
- **不追流量**：这个站的目标只有一个 —— 让招聘方 / 合作方 30 秒内 get 到我能做什么。流量、订阅、转化都不是 KPI。

## 改造范围

骨架（Hero / Projects / Writing / Now / About）已经搭好。下一步是把每个项目的 case study 慢慢补充完整 —— 不批量出货，遇到能写的就写一篇。

如果你看到这篇，说明骨架已经上线了。欢迎随便点点看。
