## 解决什么

ESP32 / Raspberry Pi 这类小硬件想接 AI 能力，每家服务商接口都不一样，鉴权 / 限流 / 容灾各做一套很烦。MASF 把它们抽象成一层统一接口，硬件端只接一次。

## 关键模块

- **统一 API gateway**：语音识别、TTS、翻译、文生图、对话补全 —— 一个 token 全部通用。
- **流式优先**：音频上行 / 文本下行都是流式，符合嵌入式设备的低延迟需求。
- **快速接入新模型**：背后接谁（OpenAI / 阿里 / 百度 / 本地模型）对硬件透明。

## 状态

底层在跑，硬件端 SDK 持续迭代。
